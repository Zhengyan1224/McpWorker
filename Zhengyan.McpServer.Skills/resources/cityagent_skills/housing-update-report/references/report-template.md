# 住房维度更新报告模板

在生成最终结果前，先按以下结构组织内容。

## 标准模板

```markdown
# 住房维度更新报告

## 一、住房问题

1. **[问题一标题]**
   [根据输入内容改写生成问题描述]

2. **[问题二标题]**
   [根据输入内容改写生成问题描述]

## 二、更新建议

1. **[建议一标题]**
   [根据输入内容改写生成建议描述]

2. **[建议二标题]**
   [根据输入内容改写生成建议描述]

## 三、立项依据

### （一）政策支持
[根据输入内容生成政策依据，如未提供具体政策名称则概括表述]

### （二）资金支持
[根据输入内容生成资金来源和支持方式，如未提供则用稳妥表述]

## 四、项目建议

1. **[项目建议一]**
   [根据输入内容改写生成项目目标、建设内容、责任单位、投资测算等内容]

2. **[项目建议二]**
   [根据输入内容改写生成项目目标、建设内容、责任单位、投资测算等内容]
```

## 生成要求

- 一级、二级标题必须完整保留
- 可以根据输入内容调整每条编号项的标题和正文
- 如果某一类信息较少，可以整合归并，但不要删掉整个章节

## 缺失信息处理

仅当输入信息不足时，允许使用以下风格的保守表达：

- 政策支持：
  - “相关工作具备一定政策支持基础，具体政策文件可在后续论证阶段进一步补充完善。”
- 资金支持：
  - “资金安排可结合后续统筹渠道进一步研究落实，具体支持方式有待进一步明确。”
- 责任单位或实施主体：
  - “相关责任分工可在项目深化实施阶段进一步明确。”
- 投资测算：
  - “具体投资规模和测算结果建议在下一步项目论证中进一步细化。”

这些句式只能作为结构补足，不应替代用户已明确提供的信息。

## 参考样例使用规则

如果任务中额外提供 `text.txt`：

- 可以参考其结构、语气、篇幅和表达颗粒度
- 不得直接复用其中的具体地区、年份、数字、政策名称、项目名称
- 除非本次输入明确给出，否则不得借用样例中的任何事实细节
